# Инструкция по добавлению нового контента на сайт

Этот документ описывает процесс добавления новых новостей, афиш или дипломов на сайт с помощью скрипта `scripts/add-content.js`.

---

## 🚀 Общий процесс

1.  **Создайте новую папку** в `scripts/staging/`. Назовите ее по названию вашего события/новости (например, `new-jazz-concert`).
2.  **Поместите в эту папку:**
    *   Один текстовый файл (`.txt`): `news.txt`, `afisha.txt` или `diplom.txt`. Выбор файла определяет тип контента.
    *   Файлы изображений (`.jpg`, `.png`, `.webp` и т.д.). Один из них может быть `cover.jpg` (обложка).
3.  **Отредактируйте текстовый файл**, заполнив все необходимые поля во `frontmatter` (метаданные) и добавив основной текст контента.
4.  **Запустите скрипт** `add-content.js` из корневой директории проекта:
    ```bash
    node scripts/add-content.js
    ```
5.  Скрипт автоматически обработает папку, переместит изображения, создаст Markdown-файл в `public/media`, обновит оба манифеста и удалит исходную папку из `scripts/staging`.

---

## 📝 Структура текстового файла (`.txt`)

Каждый текстовый файл должен иметь две основные части, разделенные `---` (тремя минусами):

*   **Frontmatter (метаданные):** Информация о контенте в формате ключ: значение.
*   **Body (основной текст):** Содержание новости/афиши, поддерживающее форматирование Markdown.

**Пример:**

```
Title: Заголовок вашего контента
Date: YYYY-MM-DD
Slug: unikalnyy-slug-dlya-url
[Дополнительные поля для Афиши]
---
Основной текст вашего контента здесь.
Используйте **Markdown** для форматирования текста.
- Пункт 1
- Пункт 2
```

---

## 🛠️ Поля Frontmatter

### Обязательные поля:

*   **`Title`**: Заголовок новости, афиши или диплома.
*   **`Date`**: Дата события или публикации в формате `YYYY-MM-DD` (например, `2026-07-29`).
*   **`Slug`**: Уникальный идентификатор для URL. Используйте латинские буквы, цифры и дефисы (например, `dzhaz-kontsert-biblioteka-2021-06-21`). *Обязательно заполните это поле!*

### Опциональные поля (только для `afisha.txt`):

*   **`Time`**: Время проведения события (например, `19:00`).
*   **`Venue`**: Место проведения события (например, `Концертный зал "Унион"`).
*   **`Tags`**: Теги, разделенные запятыми (например, `концерты, джаз, студенты`).

### Автоматически добавляемые/обрабатываемые поля:

*   **`category`**: Будет автоматически добавлена скриптом на основе имени вашего `.txt` файла (`["news"]`, `["afisha"]`, `["diplom"]`).
*   **`coverImageId`**: Будет автоматически добавлен скриптом, если вы поместите файл `cover.jpg` в папку.

---

## 📸 Изображения

*   Все `.jpg`, `.png`, `.webp` и другие поддерживаемые изображения из вашей папки будут перемещены в конечную папку медиа.
*   Если вы хотите указать изображение-обложку, назовите его `cover.jpg` (или `cover.png` и т.д.). В противном случае первое найденное изображение будет использовано как обложка.
*   Все остальные изображения автоматически станут частью галереи и будут отображаться в лайтбоксе.

---

## ⚠️ Важные примечания

*   **Даты:** Убедитесь, что формат даты `YYYY-MM-DD` соблюдается строго.
*   **Slug:** Каждый `Slug` должен быть уникальным. Если `Slug` уже существует, скрипт пропустит добавление контента.
*   **Тип контента:** Имя текстового файла (`news.txt`, `afisha.txt`, `diplom.txt`) определяет, как контент будет классифицирован на сайте.
*   **Staging:** Папка, которую обрабатывает скрипт (`scripts/staging/ваша-папка-с-контентом`), будет удалена после успешной обработки.

---

**Пример структуры папки `scripts/staging/my-new-event`:**

```
my-new-event/
├── afisha.txt
├── cover.jpg
└── image_01.jpg
└── image_02.png
```
