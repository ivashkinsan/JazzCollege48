# Руководство по управлению контентом (v3.0 - Автоматизированное добавление)

Этот документ описывает новый, значительно упрощенный процесс добавления контента (новости, афиши, дипломы) и связанных с ним медиафайлов. Используйте автоматизированный скрипт `scripts/add-content.js` для быстрой и корректной интеграции вашего контента.

---

## 1. Концепция: Автоматизированное добавление контента

Теперь весь процесс добавления новых записей (новости, афиши) вместе с изображениями максимально автоматизирован. Вам больше не нужно вручную редактировать `media-manifest.json` или перемещать файлы. Достаточно подготовить контент в специальной папке для стейджинга (`scripts/staging`), а скрипт сделает всю остальную работу:
*   Переместит изображения в центральную медиатеку (`public/media/`).
*   Обновит `src/data/media-manifest.json`.
*   Сгенерирует соответствующий Markdown-файл (`.md`) для вашей новости или афиши.

---

## 2. Рабочий процесс: Пошаговая инструкция

### Шаг 1: Подготовка контента в папке стейджинга (`scripts/staging`)

1.  **Создайте папку для стейджинга (если она еще не существует):**
    ```bash
    mkdir -p scripts/staging
    ```

2.  **Для каждой новой записи (новость, афиша, диплом) создайте отдельную подпапку** внутри `scripts/staging`.
    Например, для новой новости:
    ```
    scripts/staging/moy-pervyy-master-klass/
    ```

3.  **Внутри каждой подпапки подготовьте файлы:**

    *   **Файлы изображений:** Положите все изображения, относящиеся к этой записи.
        *   Если среди них есть **изображение-обложка**, назовите его `cover.jpg` (или `cover.png`, `cover.jpeg` и т.д.). Скрипт автоматически распознает его как обложку.
        *   Остальные изображения будут добавлены в галерею.

    *   **Один текстовый файл описания контента:**
        Этот файл определяет тип контента и содержит все необходимые метаданные и основной текст. Название файла должно соответствовать типу контента:
        *   Для **новостей**: `news.txt`
        *   Для **афиш**: `afisha.txt`
        *   Для **дипломов**: `diplom.txt` (Обратите внимание: для дипломов пока не существует отдельного лоадера в приложении, но скрипт создаст для них Markdown-файл и добавит медиа в манифест).

        **Формат файла описания:**
        Файл состоит из двух частей, разделенных строкой `---`:
        *   **Верхняя часть (метаданные):** Пары `Ключ: Значение`. Обязательные поля: `Title`, `Date`.
        *   **Нижняя часть (основной текст):** Тело Markdown-контента.

        **Примеры файлов описания:**

        **`news.txt` (для новости):**
        ```
        Title: Наши студенты на фестивале джаза
        Date: 2026-07-01
        Category: студенты
        Slug: nashi-studenty-na-festivale-dzhaza
        ---
        Описание новости о том, как наши студенты великолепно выступили на фестивале.
        Можно использовать **Markdown** для форматирования текста.
        ```
        *Поле `Slug` необязательно, если не указано, будет сгенерировано из `Title`.*
        *Поле `Category` используется для Markdown-файла новости.*

        **`afisha.txt` (для события афиши):**
        ```
        Title: Вечер фортепианного джаза
        Date: 2026-07-15
        Time: 19:00
        Venue: Большой концертный зал
        Category: концерты
        Tags: джаз, фортепиано, концерт, студенты
        ---
        Приглашаем всех ценителей джаза на незабываемый вечер фортепианной музыки.
        В программе классические композиции и современные импровизации.
        ```
        *Поля `Time`, `Venue`, `Category`, `Tags` являются необязательными, но рекомендуются для афиш.*

        **`diplom.txt` (для диплома):**
        ```
        Title: Диплом за участие в конкурсе
        Date: 2026-06-20
        Category: награды
        ---
        Этот диплом был вручен нашему студенту за выдающиеся достижения в региональном конкурсе.
        ```

### Шаг 2: Запуск автоматизации

После того как вы подготовили все подпапки с контентом в `scripts/staging`, запустите скрипт:

1.  Откройте терминал в корневой директории проекта.
2.  Выполните команду:
    ```bash
    node scripts/add-content.js
    ```

**Что происходит после запуска скрипта:**
*   Скрипт сканирует `scripts/staging` и обрабатывает каждую подпапку.
*   Для каждой записи:
    *   Все изображения из подпапки перемещаются в соответствующую по году и slug директорию внутри `public/media/`.
    *   Файл `src/data/media-manifest.json` обновляется: добавляется новый "альбом" с уникальными ID для каждого перемещенного изображения.
    *   Создается новый Markdown-файл (`.md`) в `src/news/<год>/`, `src/afisha/<год>/` или `src/diploms/<год>/` с учетом извлеченных метаданных. В этом файле автоматически прописываются `coverImageId` и `<!-- gallery -->` блок, ссылающиеся на `id` изображений из `media-manifest.json`.
    *   После успешной обработки, исходная подпапка из `scripts/staging` удаляется.

---

## 3. Резюме

Новый рабочий процесс:

1.  **Подготовьте контент:** Создайте подпапку в `scripts/staging/`, положите туда изображения (одно `cover.jpg`) и файл `news.txt`, `afisha.txt` или `diplom.txt`.
2.  **Запустите скрипт:** `node scripts/add-content.js`

Это позволяет быстро и безошибочно добавлять новый контент, обеспечивая его соответствие структуре проекта.
