# Руководство по управлению контентом (Архитектура на SQLite)

## Обзор

Проект использует централизованную базу данных **SQLite** для хранения всего контента. Этот подход заменяет старую, файловую систему, основанную на `.md` файлах и сложных `JSON-манифестах`, и обеспечивает надежность, гибкость и простоту в управлении данными.

-   **Файл базы данных**: `src/data/database.db`
-   **Управление**: Локальный API-сервер (`scripts/admin-server.ts`) и админ-панель.

---

## 1. Архитектура данных

### 1.1. Схема базы данных

База данных состоит из двух основных таблиц:

1.  **`content`**: Хранит основную информацию о каждой единице контента.
    -   `id`: `INTEGER PRIMARY KEY` - Уникальный, неизменяемый числовой идентификатор.
    -   `slug`: `TEXT` - Уникальный URL-дружественный идентификатор (например, `dzhazovyy-dzhem-seyshn`).
    -   `title`: `TEXT` - Заголовок.
    -   `date`: `TEXT` - Дата в формате `YYYY-MM-DD`.
    -   `category`: `TEXT` - Основная категория (`новость`, `афиша`, `фотоальбом`, `дипломы`).
    -   `subcategory`: `TEXT` - Опциональная подкатегория (например, для фотоальбомов).
    -   `body`: `TEXT` - Основной текст статьи в формате Markdown.
    -   `cover_image_src`: `TEXT` - Путь к изображению-обложке.
    -   `linked_photoalbum_id`: `INTEGER` - ID фотоальбома, если контент (например, новость) связан с фотоальбомом.
    -   ...и другие поля (`time`, `venue`, `tags`).

2.  **`gallery_images`**: Хранит информацию об изображениях в галереях.
    -   `id`: `INTEGER PRIMARY KEY`.
    -   `content_id`: `INTEGER` - Внешний ключ, связывающий изображение с записью в таблице `content`.
    -   `src`: `TEXT` - Путь к файлу изображения.
    -   `title`: `TEXT` - Название изображения.

### 1.2. Поток данных

#### Режим разработки

В режиме разработки (`npm run dev`) клиентская часть сайта обращается к локальному API-серверу для получения контента:

1.  **React-компонент** (например, `NewsPage.tsx`) вызывает функцию-загрузчик.
2.  **Функция-загрузчик** выполняет `fetch` запрос к локальному API-серверу (например, `fetch('http://localhost:4000/api/news')`).
3.  **API-сервер** (`scripts/admin-server.ts`) делает SQL-запрос к базе данных `database.db`.
4.  Сервер отдает данные в формате **JSON**, которые отображаются в компоненте.

#### Сборка и развертывание (Production)

Для развертывания сайта и обеспечения высокой производительности используется генерация статических JSON-файлов:

1.  Скрипт **`scripts/generate-static-data.ts`** читает данные из `database.db`.
2.  Он преобразует их в статические файлы (`public/data/news.json`, `public/data/afisha.json` и т.д.).
3.  Клиентская часть в продакшен-сборке запрашивает эти статические файлы напрямую (например, `fetch('/data/news.json')`).

Запуск `generate-static-data.ts` интегрирован в скрипты `npm run dev`, `npm run build`, `npm run admin` и `npm run deploy`, что гарантирует актуальность данных.

## 2. Управление контентом через админ-панель

Для создания, редактирования и удаления контента используется локальная админ-панель.

### 2.1. Запуск

Админ-панель доступна по адресу `http://localhost:5173/admin.html` после одновременного запуска сервера разработки и админ-сервера:

```bash
# В одном терминале
npm run dev

# В другом терминале
npm run admin
```

Админ-панель — это React-приложение, которое взаимодействует с API-сервером для всех операций с базой данных (CRUD).

### 2.2. Ключевые возможности и исправления

-   **Темный режим** по умолчанию.
-   **Корректное форматирование даты** в полях ввода.
-   **Отображение фотогалереи и библиотеки** из статических JSON файлов.
-   **Устранение дублирования фотоальбомов**: вкладка "Фотоальбомы" теперь показывает только самостоятельные альбомы.
-   **Исправление ошибок обновления контента** и улучшение UX.

### 2.3. API-эндпоинты

Сервер `admin-server.ts` предоставляет два типа эндпоинтов:

-   **Публичные (`/api/news`, `/api/afisha`)**: Используются сайтом для получения контента.
-   **Административные (`/api/content`, `/api/content/:id`)**: Используются админ-панелью для CRUD-операций.

---

## 3. Скрипты для работы с базой данных

В `package.json` доступны скрипты для управления базой данных:

-   `npm run db:setup`: Создает файл `database.db` и пустые таблицы.
-   `npm run db:cleanup`: Стандартизирует значения в колонке `category`.
-   `npm run db:add-subcategory`: Добавляет колонку `subcategory` в таблицу `content`.
-   `npm run db:migrate`: (Выполнялся один раз) Переносит контент из старой файловой системы в базу данных SQLite.

Эта система обеспечивает надежное и профессиональное управление контентом проекта.
