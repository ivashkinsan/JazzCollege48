# 09-data-parsing.md

## Обзор логики парсинга данных для новостей и афиши

В данном проекте данные для разделов "Новости" и "Афиша" хранятся в формате Markdown-файлов с использованием так называемого "frontmatter" – блока метаданных в формате YAML, заключенного между двумя строками `---`.

Основная логика парсинга реализована в файле `src/data/dataLoaders.ts`.

### 1. Загрузка файлов (`import.meta.glob`)

Для динамической загрузки всех Markdown-файлов из соответствующих директорий (`src/news/**` и `src/afisha/**`) используется функция `import.meta.glob` от Vite. Она позволяет импортировать файлы как "сырые" строки, что критично для последующего парсинга:

```typescript
const newsModules = import.meta.glob('../news/**/*.md', { query: '?raw', import: 'default' });
const afishaModules = import.meta.glob('../afisha/**/*.md', { query: '?raw', import: 'default' });
```

Каждый элемент `newsModules` или `afishaModules` представляет собой функцию, которая при вызове (`await newsModules[path]()`) возвращает содержимое соответствующего Markdown-файла в виде строки.

### 2. Функция `parseMarkdown(content: string)`

Эта функция является сердцем механизма парсинга. Ее задача – разделить переданную строку Markdown на две части: блок метаданных (frontmatter) и основное содержимое (body).

#### 2.1. Удаление BOM (Byte Order Mark)

Перед началом парсинга функция удаляет возможный невидимый символ BOM (`\uFEFF`) из начала строки:

```typescript
const cleanedContent = content.replace(/^\uFEFF/, '');
```

Это необходимо, поскольку некоторые редакторы могут добавлять BOM в начало UTF-8 файлов, что может помешать корректной работе регулярных выражений, ожидающих `---` в самом начале.

#### 2.2. Извлечение Frontmatter и Body

Используется регулярное выражение для разделения контента на frontmatter и body:

```typescript
const frontmatterMatch = cleanedContent.match(new RegExp(`^---([\s\S]*?)---([\s\S]*)$`));
```

-   `^---`: Соответствует начальной строке `---`.
-   `([\s\S]*?)`: Нежадный захват всего содержимого между первой и второй `---`. Это и есть строка frontmatter.
-   `---`: Соответствует закрывающей строке `---`.
-   `([\s\S]*)`: Захватывает все оставшееся содержимое после второго `---`. Это и есть тело документа (body).

Если шаблон не найден (например, файл не содержит блока frontmatter), функция возвращает `null`.

#### 2.3. Парсинг Frontmatter

Извлеченная строка `frontmatterStr` затем разбивается на отдельные строки:

```typescript
const lines = frontmatterStr.split(/?
/);
```

Использование `/?
/` обеспечивает корректную обработку как Unix (`
`), так и Windows (`
`) окончаний строк.

Затем каждая строка frontmatter парсится для извлечения пар "ключ: значение":

```typescript
for (const line of lines) {
    const match = line.match(/^([^:]+):\s*(.*)$/);
    if (match) {
        const key = match[1].trim();
        const value = match[2].trim().replace(/^["']|["']$/g, ''); // Удаление кавычек
        frontmatter[key] = value;
    }
}
```

Ключи и значения обрезаются от пробелов. Значения также очищаются от внешних кавычек (одинарных или двойных).

#### 2.4. Возвращаемое значение

Функция возвращает объект `{ frontmatter: Record<string, string>, body: string }`, где `frontmatter` – это объект с ключами и значениями метаданных, а `body` – это основное содержимое Markdown.

### 3. Функция `parseGallery(content: string)`

Эта вспомогательная функция предназначена для извлечения путей к изображениям галереи, которые могут быть указаны в Markdown-файле в специальном блоке комментариев:

```markdown
<!-- gallery -->
- /path/to/image1.jpg
- /path/to/image2.jpg
```

Логика:
1.  Используется регулярное выражение для поиска блока `<!-- gallery -->` и всего содержимого после него.
2.  Извлеченное содержимое разбивается на строки с помощью `split(/?
/)`.
3.  Каждая строка очищается (удаляется `-` и пробелы), фильтруется от пустых значений и преобразуется в полный путь к активу с помощью функции `asset()`.

### 4. Функции `loadNews()` и `loadAfisha()`

Эти асинхронные функции:
-   Итерируют по всем загруженным `newsModules` или `afishaModules`.
-   Для каждого модуля получают сырое содержимое файла.
-   Передают содержимое в `parseMarkdown` для извлечения метаданных и тела.
-   Извлекают необходимые поля (`title`, `date`, `category`, `cover`, `time`, `venue`, `tags`) из `frontmatter`.
-   Формируют объект `ExtendedNewsItem` или `AfishaItem`.
-   Используют `parseGallery` для обработки изображений галереи.
-   Сохраняют загруженные элементы в глобальные массивы `news` и `afisha` и сортируют их по дате.
-   Включают механизм кэширования для предотвращения повторной загрузки данных.

Эта архитектура позволяет легко добавлять новый контент, просто создавая Markdown-файлы в соответствующих директориях, без необходимости изменения кода приложения.
