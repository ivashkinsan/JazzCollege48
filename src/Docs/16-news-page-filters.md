# Документация: Компонент фильтров на странице новостей

Этот документ описывает финальную реализацию и поведение блока фильтров на странице "Новости и события" (`src/pages/NewsPage.tsx`).

## 1. Общая структура

После рефакторинга и объединения разделов "Новости" и "Фотогалерея" был создан единый блок фильтрации, который включает:
- Общий заголовок "Фильтры".
- **Два независимых, горизонтально прокручиваемых ряда кнопок**:
  1.  Фильтр по годам.
  2.  Фильтр по категориям ("концерты", "мастер-классы", "конкурсы", "разное").

## 2. Ключевые механики и их реализация

### 2.1. "Прилипание" (Sticky) блока фильтров

- **Задача:** Сделать так, чтобы весь блок с фильтрами "прилипал" к шапке сайта при вертикальной прокрутке страницы вниз.
- **Реализация:**
  - Контейнеру `.filtersSection` заданы свойства `position: sticky` и `top: var(--header-height)`.
  - Для решения распространенных проблем с рендерингом `sticky-`элементов, которые находятся внутри других контейнеров, добавлено свойство `will-change: transform;`. Оно "подсказывает" браузеру, что элемент будет меняться, и выносит его на отдельный слой отрисовки, что стабилизирует поведение.

### 2.2. Горизонтальная прокрутка рядов с кнопками

- **Задача:** Обеспечить, чтобы каждый ряд кнопок можно было свайпать (скроллить) по горизонтали, если он не помещается в ширину экрана, и чтобы этот скролл не затрагивал всю страницу.
- **Реализация:**
  - Контейнер для метки и ряда кнопок (`.filterGroup`) использует `display: grid` с `grid-template-columns: auto minmax(0, 1fr);`. Эта конструкция заставляет второй элемент (ряд с кнопками) корректно обрабатывать переполнение.
  - Контейнер, непосредственно оборачивающий кнопки (`.buttonRow`), имеет свойство `overflow-x: auto`.
  - Для предотвращения "утечки" скролла на основную страницу (особенно на мобильных устройствах) используется `overscroll-behavior-x: contain;`.
  - Скроллбар скрыт для более чистого вида.

### 2.3. Визуальная подсказка для скролла

- **Задача:** Интуитивно показать пользователю, что ряд можно прокручивать.
- **Реализация:**
  - Для каждого ряда кнопок используется дополнительная обертка `.buttonRowWrapper`.
  - С помощью псевдо-элемента `::after` на эту обертку накладывается полупрозрачный градиент с правого края. Он создает эффект "затухания", намекая, что за видимой областью есть еще контент.

### 2.4. Центрирование активного фильтра при клике

- **Задача:** При выборе фильтра (клике по кнопке) плавно прокрутить ряд так, чтобы активная кнопка оказалась в центре видимой области.
- **Реализация:**
  - В компоненте `NewsPage.tsx` создан обработчик `handleFilterClick`.
  - При клике на кнопку вызывается этот обработчик, который получает DOM-элемент кнопки.
  - На элементе вызывается нативный метод браузера `scrollIntoView()` с параметрами `{ behavior: 'smooth', inline: 'center', block: 'nearest' }`. Это обеспечивает плавную и корректную прокрутку.
