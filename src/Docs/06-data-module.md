# 6. Модуль данных

## Обзор

Модуль данных (`src/data`) отвечает за предоставление всей информации для приложения. После рефакторинга его архитектура была значительно упрощена и централизована вокруг базы данных **SQLite**.

- **База данных** (`src/data/database.db`): Единственный источник правды для всего динамического контента (новости, афиша и т.д.).
- **Загрузчики данных** (`src/data/newsLoader.ts`): Функции для асинхронной загрузки данных **из API**, а не из файлов.
- **API-сервер** (`scripts/admin-server.ts`): Сервер, который инкапсулирует всю логику работы с базой данных.
- **Статические данные** (`src/data/static`): Данные, которые не меняются (список преподавателей и т.д.), остались без изменений.
- **Типы** (`src/types`): Интерфейсы TypeScript, описывающие структуру всех данных.

---

## Структура модуля `src/data`

- `src/data/`
  - `database.db`: Файл базы данных SQLite.
  - `index.ts`: Главный файл, который реэкспортирует функции-загрузчики.
  - `newsLoader.ts`: **(Ключевое изменение)** Содержит функции `loadNews` и `loadAfisha`, которые теперь выполняют `fetch`-запросы к API-серверу.
  - `static/`: Папка со статическими данными (не изменилась).
    - `videos.ts`, `teachers.ts`, `navigation.ts`...

## Загрузка динамических данных (Новый подход)

Логика загрузки была полностью переработана для надежности и производительности.

**Процесс устроен следующим образом:**

1.  **API-сервер**: Сервер `scripts/admin-server.ts` имеет прямой доступ к `database.db`. Он предоставляет публичные эндпоинты `/api/news` и `/api/afisha`.

2.  **Загрузчики (`newsLoader.ts`)**:
    - При вызове, загрузчик (например, `loadNews()`) выполняет `fetch`-запрос к соответствующему эндпоинту (`http://localhost:4000/api/news`).
    - Он получает готовый `JSON` с данными, уже отсортированными и отформатированными на стороне сервера.
    - Результаты кешируются в памяти, чтобы избежать повторных сетевых запросов при навигации по сайту.

### `newsLoader.ts`
- **`loadNews(): Promise<ExtendedNewsItem[]>`**: Асинхронно запрашивает данные с эндпоинта `/api/news`.
- **`loadAfisha(): Promise<AfishaItem[]>`**: Аналогично запрашивает данные с эндпоинта `/api/afisha`.

### Использование в компонентах
Импорт и использование в компонентах не изменились. Компоненты по-прежнему вызывают `loadNews()` и обрабатывают полученный `Promise`.

```typescript
import { loadNews } from '../data'; // Импорт из папки data

const [newsData, setNewsData] = useState<ExtendedNewsItem[]>([]);

useEffect(() => {
  loadNews().then(setNewsData);
}, []);
```

### Адаптация данных для достижений
Для данных типа `achievements`, получаемых с API-сервера, выполняется клиентская адаптация. Это необходимо, так как сервер возвращает имена полей в `snake_case` (например, `student_name`, `image_src`), тогда как в клиентском интерфейсе `Achievement` используются имена в `camelCase` (`studentName`, `image`). Также обрабатывается новое поле `city`.
-   **Поля `student_name`, `image_src`, `city`**: Преобразуются в `studentName`, `image`, `city` соответственно.
-   **Реализация**: Подробности адаптации можно найти в функции `fetchItems` компонента `AdminApp.tsx` (файл `src/admin/AdminApp.tsx`).

Этот подход полностью разделяет клиентскую часть (которая только запрашивает и отображает данные) и серверную (которая работает с базой данных).
