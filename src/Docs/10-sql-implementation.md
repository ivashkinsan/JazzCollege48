# SQL Implementation (Будущая реализация SQL)

## 1. Введение
Данный документ описывает планы и подходы к интеграции реляционной базы данных (предположительно MySQL) в проект JazzCollege48. Основная цель — централизованное и структурированное хранение динамического контента сайта, такого как новости, афиши, информация о преподавателях, выпускниках и т.д.

## 2. Мотивация
*   **Динамический контент:** Возможность легкого обновления и добавления контента без изменения кода.
*   **Управление данными:** Централизованное управление всей текстовой и ссылочной информацией.
*   **Масштабируемость:** Подготовка к росту объема данных и функциональности.
*   **Поиск и фильтрация:** Облегчение реализации функций поиска, фильтрации и сортировки контента.

## 3. Выбор СУБД
Предполагается использование **MySQL**, так как эта СУБД доступна на хостинге Reg.ru и является широко распространенным решением для веб-приложений.

## 4. Области применения
SQL база данных будет использоваться для хранения следующего типа контента:
*   **Новости:** Заголовки, текст, даты публикации, ссылки на изображения, категории.
*   **Афиши/Мероприятия:** Названия, описания, даты, время, место проведения, ссылки на изображения.
*   **Преподаватели:** ФИО, должности, описания, ссылки на фотографии.
*   **Выпускники:** ФИО, год выпуска, достижения, ссылки на фотографии.
*   **Документы:** Названия документов, описания, ссылки на PDF/TXT файлы.
*   **Пользователи (админ-панель):** Учетные данные для доступа к системе управления контентом.

## 5. Общие принципы хранения данных
*   **Текстовый контент:** Будет храниться непосредственно в базе данных (например, в полях типа `TEXT` или `VARCHAR`).
*   **Медиа-контент (изображения, PDF):** Сами файлы будут храниться на файловой системе сервера (в папке `public`). В базе данных будут храниться только пути (URL) к этим файлам. Это обеспечивает высокую производительность и эффективное использование ресурсов базы данных.

## 6. Примеры таблиц (предварительно)

### Таблица `news` (Новости)
```sql
CREATE TABLE news (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    slug VARCHAR(255) UNIQUE NOT NULL, -- Для читаемых URL
    content TEXT,
    image_url VARCHAR(255), -- Путь к изображению новости
    category VARCHAR(100),
    publication_date DATETIME DEFAULT CURRENT_TIMESTAMP,
    is_published BOOLEAN DEFAULT TRUE
);
```

### Таблица `events` (Афиши/Мероприятия)
```sql
CREATE TABLE events (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    event_date DATETIME NOT NULL,
    location VARCHAR(255),
    image_url VARCHAR(255), -- Путь к изображению афиши
    ticket_link VARCHAR(255),
    is_active BOOLEAN DEFAULT TRUE
);
```

## 7. Интеграция с приложением
*   **API:** Будет разработан API (Application Programming Interface) на бэкенде (например, с использованием Node.js/Express или другого фреймворка), который будет взаимодействовать с базой данных.
*   **Фронтенд:** React-приложение будет отправлять запросы к этому API для получения и отображения данных.
