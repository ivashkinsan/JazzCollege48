# Документация: Архитектура роутинга API админ-панели

Этот документ описывает финальную архитектуру API для получения списков данных в административной панели.

## 1. Проблема

В ходе рефакторинга возникла ситуация, когда запросы на получение списков для разделов "Достижения", "Выпускники" и т.д. стали возвращать ошибку `404 Not Found`.

Причиной послужил конфликт маршрутов в `ContentController`, который изначально пытался обрабатывать запросы и для контента (`news`, `afisha`), и для других сущностей (`achievements` и т.д.) через один и тот же параметризованный эндпоинт `@Get('admin/list/:manager')`. Попытка исправить этот конфликт сделала эндпоинт слишком специфичным, что "сломало" загрузку для всех остальных разделов.

## 2. Финальная архитектура

Для устранения этой проблемы и создания более чистой и предсказуемой архитектуры, логика была разделена на два независимых модуля.

### 2.1. `ContentModule`

- **Ответственность:** Управляет только сущностями, которые хранятся в общей таблице `content` (на данный момент это `news` и `afisha`).
- **Контроллер (`ContentController`):** Содержит **явный** эндпоинт без параметров для получения списка всех записей контента:
  - `GET /api/admin/list/content`
- **Клиентская логика (`store.ts`):** При активной вкладке "Новости" или "Афиша", клиент запрашивает этот общий эндпоинт, а затем самостоятельно фильтрует полученный массив по полю `category`, чтобы отобразить нужные данные.

### 2.2. `AdminModule` (Новый)

- **Ответственность:** Управляет запросами на получение списков для всех **остальных** разделов админ-панели, где каждая сущность имеет свою собственную таблицу в БД (`achievements`, `graduates`, `videos`, `library_links`).
- **Контроллер (`AdminController`):** Содержит **параметризованный** эндпоинт:
  - `GET /api/admin/list/:manager`
- **Сервис (`AdminService`):** Содержит логику, которая по параметру `:manager` определяет, из какой именно таблицы (`achievements`, `graduates` и т.д.) нужно сделать запрос.
- **Клиентская логика (`store.ts`):** Для всех вкладок, кроме "Новостей" и "Афиш", клиент вызывает этот эндпоинт, подставляя в URL ключ активной вкладки (например, `GET /api/admin/list/graduates`).

Этот подход полностью устраняет конфликты маршрутизации и делает архитектуру бэкенда более модульной и масштабируемой.
